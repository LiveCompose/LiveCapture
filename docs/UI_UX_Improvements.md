# LiveCapture UI/UX 优化说明文档

## 概述
本次更新对 LiveCapture 应用的用户界面和用户体验进行了全面优化，提升了易用性、视觉效果和交互体验。

## 主要优化内容

### 1. 右上角菜单优化和丰富 ✅

#### 优化前
- 仅有简单的"显示/隐藏调试模式"选项

#### 优化后
菜单结构更加完善，包含以下功能模块：

**调试功能**
- 显示/隐藏调试信息

**相机设置**（子菜单）
- 切换镜头（前置/后置）
- 锁定焦点（预留接口，待实现）

**拍摄设置**（子菜单）
- 开启/关闭自动拍照
- 拍照延迟设置（0.5秒、1.0秒、1.5秒、2.0秒可选）
- 当前延迟时间显示

**帮助和关于**
- 使用帮助（预留接口）
- 关于页面（预留接口）

**特点：**
- 使用层级菜单组织，结构清晰
- 使用 SF Symbols 图标增强视觉识别
- 实时显示当前设置状态（如延迟时间、自动拍照开关）
- 已选中项显示勾选标记

---

### 2. 用户引导文字系统 ✅

#### 功能描述
在相机预览上边框下方添加动态用户引导文字，根据应用不同阶段提供清晰的操作指引。

#### 实现位置
- 位于顶部控制栏下方
- 独立于调试信息显示
- 使用胶囊形状的半透明背景
- 白色边框和阴影效果，确保在各种背景下清晰可见

#### 引导文字内容（随状态变化）
| 阶段 | 引导文字 |
|------|---------|
| 空闲 | （无） |
| 启动相机 | "正在启动相机..." |
| 等待稳定 | "请保持设备稳定" |
| 检测区域 | "正在识别目标区域..." |
| 模板就绪 | "请将中心点移动到目标位置" |
| 对齐中 | "继续对准中心..." |
| 准备拍照 | "请保持稳定，即将拍照" |
| 正在拍照 | "正在拍照..." |
| 保存照片 | "照片已保存" |
| 错误 | "发生错误，请重试" |

#### 视觉特点
- 平滑的淡入淡出动画
- 从顶部滑入的过渡效果
- 半透明黑色背景（60%不透明度）
- 白色文字，确保可读性
- 圆角胶囊形状设计

---

### 3. 调试模式重新设计 ✅

#### 优化前
- 简单的列表式显示
- 信息密集且难以快速查看
- 视觉层次不清晰

#### 优化后

**视觉设计**
- 深色半透明卡片背景（75%不透明度）
- 橙色渐变边框（品牌色）
- 阴影效果增强层次感
- 分组显示，使用分隔线区分不同类别

**信息组织**
采用图标 + 标题 + 数值的三列布局：

**标题栏**
- 🐜 调试信息标题（橙色图标）
- 关闭按钮

**主要状态信息**
- ⚙️ 状态：当前处理阶段
- 🎯 稳定性：稳定/不稳定（带颜色指示）

**跟踪和检测信息**
- 🎯 跟踪位置：(x, y) 坐标
- ↔️ 距离中心：像素距离（近距离显示绿色）
- ✓ 检测状态：就绪/未就绪

**相机参数**
- 📷 变焦：倍率 / 焦距
- 🎯 对准状态：已对准/未对准（带颜色指示）

**交互优化**
- 平滑的展开/收起动画
- 从顶部滑入/滑出效果
- 等宽字体显示数值，便于查看变化

**自动刷新逻辑**
- 检测到大幅度移动（超过30像素）时立即重置检测状态
- 避免用户在移动设备后仍显示旧的检测结果
- 提高跟踪精度和用户体验

---

### 4. 对准后拍照逻辑优化 ✅

#### 核心改进

**延迟机制**
- 可配置的拍照延迟（默认1.0秒，可选0.5/1.0/1.5/2.0秒）
- 对准成功后进入"准备拍照"状态
- 延迟期间显示"请保持稳定，即将拍照"提示
- 如果失去对准，自动取消拍照计划

**用户提醒**
- 新增 `readyToCapture` 状态
- 引导文字实时更新："请保持稳定，即将拍照"
- 进度条显示当前阶段（92%）
- 视觉反馈清晰明确

**缩放拍照功能**

1. **接口预留**
   - 预留了基于裁切框相对画面比例的缩放接口
   - 当前设置为固定 2x 变焦

2. **拍照动效**
   - **闪光效果**：白色全屏闪光（0.8透明度）
     - 0.1秒淡入
     - 0.2秒淡出
   - **缩放动效**：画面从 1x 放大到 2x
     - 使用弹簧动画（Spring Animation）
     - 响应时间：0.4秒
     - 阻尼系数：0.7（自然的弹性效果）
   - **恢复动效**：0.5秒后恢复到原始大小
     - 相同的弹簧动画参数
     - 平滑过渡

3. **动效触发机制**
   ```swift
   // 回调函数触发
   viewModel.onCaptureTriggered = {
       triggerCaptureAnimation()
   }
   ```

4. **视觉反馈流程**
   ```
   对准成功 → 延迟倒计时 → 闪光 → 缩放到2x → 拍照 → 恢复原始大小
   ```

**自动拍照开关**
- 新增 `isAutoCaptureEnabled` 属性
- 可在菜单中开启/关闭
- 关闭后不会自动拍照，用户可手动点击拍照按钮

## 技术实现细节

### 状态管理
```swift
// 新增状态
@Published private(set) var userGuidanceText: String = ""
@Published var isAutoCaptureEnabled: Bool = true
@Published var captureDelay: Double = 1.0
```

### 动画参数
```swift
// 拍照缩放动画
.animation(.spring(response: 0.3, dampingFraction: 0.6), value: captureAnimationScale)

// 用户引导文字动画
.animation(.spring(response: 0.4, dampingFraction: 0.8), value: guidance)

// 调试面板动画
.transition(.asymmetric(
    insertion: .move(edge: .top).combined(with: .opacity),
    removal: .move(edge: .top).combined(with: .opacity)
))
```

### 大幅度移动检测
```swift
private let stableMovementThreshold: CGFloat = 30.0

// 在 bindMotion() 中检测
let movement = sqrt(dx * dx + dy * dy)
if movement > self.stableMovementThreshold && self.detectionReady {
    self.resetDetectionState()
}
```

## 用户体验流程

### 完整拍照流程
1. **启动应用** → "正在启动相机..."
2. **相机就绪** → "请保持设备稳定"
3. **设备稳定** → "正在识别目标区域..."
4. **检测完成** → "请将中心点移动到目标位置"
5. **开始对准** → "继续对准中心..."
6. **对准成功** → "请保持稳定，即将拍照"（进入延迟倒计时）
7. **延迟结束** → 闪光 + 缩放动效
8. **拍照完成** → "照片已保存" + 恢复原始大小

### 异常处理
- **失去对准**：取消拍照计划，返回"请将中心点移动到目标位置"
- **大幅度移动**：自动重置检测，返回"请保持设备稳定"
- **设备不稳定**：暂停检测，显示"请保持设备稳定"

## 视觉风格统一

### 颜色方案
- **主色调**：黑色背景，白色前景
- **强调色**：橙色（调试模式、警告信息）
- **成功色**：绿色（稳定、对准、距离近）
- **半透明**：60-75%不透明度的背景

### 图标使用
- 统一使用 SF Symbols
- 图标大小根据上下文调整（14-20pt）
- 配合文字标签使用

### 动画原则
- 使用弹簧动画（Spring）实现自然的过渡
- 响应时间控制在 0.3-0.4 秒
- 阻尼系数 0.6-0.8，避免过度弹跳
